# เอกสารความต้องการทางธุรกิจ (Business Requirement Document - BRD)

## ชื่อโปรเจกต์: NotiKeeper (ระบบเก็บถาวรการแจ้งเตือนและบทสนทนาบนมือถือ Android สำหรับเจ้าของอุปกรณ์)

**เวอร์ชัน:** 1.0 (เอกสารฉบับแรก — ออกหลัง release v1.6 ของผลิตภัณฑ์)
**สถานะ:** ใช้งานจริง (In Production / Personal Use)
**ผู้ดูแลผลิตภัณฑ์:** Freshair129

## 1. บทสรุปผู้บริหาร (Executive Summary)

NotiKeeper คือเครื่องมือส่วนตัวบน Android สำหรับ **ทวงคืนอำนาจในการเก็บข้อมูลของผู้ใช้** จากแพลตฟอร์มแชตและการแจ้งเตือนของระบบ ระบบจะดักจับการแจ้งเตือนทุกแอป และอ่านบทสนทนาในแอปแชตหลัก (Messenger, LINE, Instagram, WhatsApp, Telegram) บันทึกลงฐานข้อมูลในเครื่องแบบเข้ารหัส เพื่อให้เจ้าของอุปกรณ์ค้นย้อนหลังได้ไม่จำกัดเวลา แม้ข้อความต้นทางจะถูกลบหรือล่วงพ้นกรอบ 24 ชั่วโมงของระบบแจ้งเตือนไปแล้ว

โครงการเป็น **personal-use tool** ไม่ใช่ผลิตภัณฑ์เชิงพาณิชย์ ไม่มีเป้าหมายรายได้ ไม่มี user acquisition และจงใจไม่เผยแพร่ผ่าน Play Store เนื่องจากธรรมชาติของฟังก์ชัน

## 2. ที่มาและโอกาสทางธุรกิจ (Business Context & Opportunity)

### 2.1 ปัญหาเชิงโครงสร้างที่ขับเคลื่อนโครงการ

| ปัญหา | ผลกระทบต่อเจ้าของข้อมูล |
| :---- | :---- |
| **ระบบแจ้งเตือน Android เก็บประวัติแค่ ~24 ชั่วโมง** | ข้อความสำคัญที่ไม่ทันอ่านหายถาวรถ้าไม่เปิดอ่านในวัน |
| **แชต Messenger ถ้าเผลอลบ จะหายจากฝั่งผู้ใช้ทันที** | ข้อมูลที่อยู่/เบอร์/ราคา/นัดหมาย ที่ฝังในแชตอาจสูญหายโดยกู้ไม่ได้ |
| **iOS ไม่มีประวัติแจ้งเตือนเลย** | ผู้ใช้ในกลุ่มเพื่อน/ครอบครัว Android เป็นพื้นที่เดียวที่ยังทำได้ |
| **Facebook "Download Your Information" ไม่ใช่ real-time** | ใช้สำหรับ archive ทั่วไป แต่ไม่ทันเหตุการณ์ และใช้ไม่ได้กับข้อความที่ลบไปแล้ว |
| **แอป "กู้แชท" บน Play Store ส่วนใหญ่หลอกลวง** | ผู้ใช้ที่หาทางออกถูกเก็บค่าใช้จ่ายโดยไม่ได้แชตคืน |

### 2.2 โอกาส (Why now)

- Android 11+ มี `NotificationListenerService` และ `AccessibilityService` ที่ทำงานเสถียร เปิดให้แอปดักจับข้อมูลที่แสดงผลในเครื่องตัวเองได้
- SQLCipher เป็น native library ที่เข้ารหัส SQLite ทั้งไฟล์ในระดับ AES-256 ไม่มีต้นทุน license
- Android Keystore + EncryptedSharedPreferences ทำให้การเก็บกุญแจปลอดภัยพอที่จะใช้กับข้อมูลส่วนตัวจริง
- เครื่องมือ Android build (JDK/Gradle/SDK) ติดตั้ง portable ได้ โดยไม่ต้องลง Android Studio

## 5. ขอบเขตทางธุรกิจ (Business Scope)

### 5.1 อยู่ในขอบเขต (In Scope)
- บัญชี/อุปกรณ์/ข้อมูลของเจ้าของเครื่องเท่านั้น
- แชตที่เจ้าของเครื่องเป็นคู่สนทนาเอง
- การ archive, ค้นหา, อ่านออกเสียง, สำรอง, อัปโหลดเข้า server ส่วนตัว
- การกระจาย APK เฉพาะ sideload + GitHub Releases

### 5.2 ออกนอกขอบเขต (Out of Scope)
- การติดตั้งบนเครื่องของบุคคลอื่นโดยไม่ได้รับความยินยอม
- การเผยแพร่/ขายข้อความของคู่สนทนา
- การขึ้น Play Store เป็นแอปสาธารณะ
- บริการ SaaS / multi-tenant
- การโฆษณา / monetization

## 7. ข้อจำกัด (Constraints)

### 7.1 ข้อจำกัดเชิงกฎหมายและนโยบาย
- **Google Play Policy:** การใช้ `AccessibilityService` เพื่อ archive เนื้อหาเป็นการใช้นอกขอบเขตของ Play → ผลิตภัณฑ์นี้ **ขึ้น Play Store ไม่ได้** ต้อง sideload เท่านั้น
- **Meta ToS:** มีข้อห้ามการดูดข้อมูลอัตโนมัติ การอ่านจอ "ของบัญชีตัวเอง" จัดอยู่ในพื้นที่สีเทา → จำกัดใช้งานส่วนตัวกับข้อมูลตัวเองเท่านั้น
- **PDPA/ความเป็นส่วนตัวคู่สนทนา:** ข้อความที่เก็บมีคำพูดของบุคคลอื่นปนอยู่ → ห้ามเผยแพร่ / ใช้กดดัน / ใช้เป็นหลักฐานสาธารณะโดยไม่มีฐานทางกฎหมาย

### 7.2 ข้อจำกัดเชิงเทคนิค
- รองรับเฉพาะ **Android 11 (API 30) ขึ้นไป** (เพื่อระบบล็อกหน้าจอที่เสถียร)
- โหมดอ่านหน้าจอเก็บได้เฉพาะ "สิ่งที่ผ่านสายตา" ไม่สามารถดึงประวัติจากเซิร์ฟเวอร์ของแอปต้นทาง
- กุญแจเข้ารหัสผูกกับ Keystore ของเครื่อง → factory reset = ข้อมูลเดิมอ่านไม่ได้ (ตามเจตนา)

### 7.3 ข้อจำกัดเชิงทรัพยากร
- ทีมพัฒนา: 1 คน (Owner) + AI assistance
- งบ: ฟรี (ไม่มี cloud service หรือ paid SDK)
